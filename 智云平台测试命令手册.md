# 智云平台测试命令手册

## 概述

本文档用于智云平台下发命令测试硬件节点响应。

- **数据流**：智云平台 → Coordinator(协调器) → ZigBee → Sensor节点 → 执行动作
- **命令格式**：`{key=value}` 或 `{"key":"value"}`
- **查询命令**：值为 `?` 表示查询，节点返回当前值

---

## 1. sensor-a — 环境采集节点

**节点地址**：`0x0001`  
**节点名称**：601  
**功能**：温湿度 + 光照采集

### 1.1 查询命令

| 说明 | 命令 | 返回示例 |
|------|------|----------|
| 查询温度 | `{temp=?}` | `{temp:27.0}` |
| 查询湿度 | `{humi=?}` | `{humi:65.0}` |
| 查询光照 | `{lux=?}` | `{lux:200}` |
| 查询上报周期 | `{V0=?}` | `{V0:30}` |
| 查询传感器使能 | `{D0=?}` | `{D0:255}` |

### 1.2 设置命令

| 说明 | 命令 | 效果 |
|------|------|------|
| 设置上报周期(秒) | `{V0=60}` | 改为60秒上报一次 |
| 设置上报周期(秒) | `{V0=10}` | 改为10秒上报一次 |
| 关闭温度上报 | `{CD0=1}` | D0 bit0 清零 |
| 开启温度上报 | `{OD0=1}` | D0 bit0 置1 |
| 关闭湿度上报 | `{CD0=2}` | D0 bit1 清零 |
| 开启湿度上报 | `{OD0=2}` | D0 bit1 置1 |
| 关闭光照上报 | `{CD0=4}` | D0 bit2 清零 |
| 开启光照上报 | `{OD0=4}` | D0 bit2 置1 |

### 1.3 自动上报字段

节点定时上报以下字段：

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `temp` | 温度(℃) | 27.0 |
| `humi` | 湿度(%RH) | 65.0 |
| `lux` | 光照(lux) | 200 |

## 2. sensor-b — 控制执行节点

**节点地址**：`0x0002`  
**节点名称**：602  
**功能**：RGB灯 + 蜂鸣器 + 继电器(门锁)

### 2.1 远程开门

| 说明 | 命令 | 效果 |
|------|------|------|
| 开门 | `{unlock=1}` | 继电器开 → 绿灯亮 → 3秒后自动关门 |
| 关门 | `{unlock=0}` | 继电器关 |

### 2.2 蜂鸣器控制

| 说明 | 命令 | 效果 |
|------|------|------|
| 门铃响 | `{buzz=500}` | 蜂鸣器短响500ms + 蓝灯亮5秒 |
| 长响 | `{buzz=1}` | 蜂鸣器持续响 |
| 关闭 | `{buzz=0}` | 蜂鸣器停 + 静音 |
| 自定义短响 | `{buzz=200}` | 蜂鸣器响200ms |

### 2.3 RGB灯控制

| 说明 | 命令 | 效果 |
|------|------|------|
| 红灯 | `{rgb=[255,0,0]}` | 亮红灯 |
| 绿灯 | `{rgb=[0,255,0]}` | 亮绿灯 |
| 蓝灯 | `{rgb=[0,0,255]}` | 亮蓝灯 |
| 黄灯 | `{rgb=[255,255,0]}` | 亮黄灯 |
| 紫灯 | `{rgb=[255,0,255]}` | 亮紫灯 |
| 青灯 | `{rgb=[0,255,255]}` | 亮青灯 |
| 白灯 | `{rgb=[255,255,255]}` | 亮白灯 |
| 关灯 | `{rgb=[0,0,0]}` | 关闭RGB |

### 2.4 告警联动

| 说明 | 命令 | 效果 |
|------|------|------|
| 安全模式 | `{alert=0}` | 绿灯 |
| 注意模式 | `{alert=1}` | 蓝灯 |
| 报警模式 | `{alert=2}` | 红灯 + 蜂鸣器长响 |
| 解除告警 | `{reset=1}` | 恢复安全状态 + 绿灯 |

### 2.5 查询命令

| 说明 | 命令 | 返回示例 |
|------|------|----------|
| 查询门锁状态 | `{unlock=?}` | `{unlock:0}` |
| 查询蜂鸣器状态 | `{buzz=?}` | `{buzz:0}` |
| 查询RGB状态 | `{rgb=?}` | `{rgb:[0,255,0]}` |
| 查询告警等级 | `{alert=?}` | `{alert:0}` |
| 查询上报周期 | `{V0=?}` | `{V0:30}` |

### 2.6 自动上报字段

节点定时上报以下字段：

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `unlock` | 门锁状态(0=锁定, 1=解锁) | 0 |
| `buzz` | 蜂鸣器状态(0=关, 1=长响, 2=短响) | 0 |
| `rgb` | RGB灯状态 [R,G,B] | [0,255,0] |
| `alert` | 告警等级(0=安全, 1=注意, 2=报警) | 0 |

## 3. sensor-c — 安防检测节点

**节点地址**：`0x0003`  
**节点名称**：603  
**功能**：PIR + 振动 + 门磁 + 火焰 + 气体 + 光栅

### 3.1 查询命令

| 说明 | 命令 | 返回示例 |
|------|------|----------|
| 查询PIR(人体红外) | `{pir=?}` | `{pir:0}` |
| 查询门铃/振动 | `{tch=?}` | `{tch:0}` |
| 查询门磁(霍尔) | `{door=?}` | `{door:0}` |
| 查询火焰 | `{flame=?}` | `{flame:0}` |
| 查询可燃气体 | `{A4=?}` | `{A4:0}` |
| 查询光栅 | `{A5=?}` | `{A5:0}` |
| 查询告警等级 | `{alert=?}` | `{alert:0}` |
| 查询停留时间 | `{stay=?}` | `{stay:0}` |
| 查询夜间状态 | `{night=?}` | `{night:0}` |
| 查询上报周期 | `{V0=?}` | `{V0:30}` |

### 3.2 安防控制命令

| 说明 | 命令 | 效果 |
|------|------|------|
| 强制夜间模式 | `{night=1}` | 启用夜间告警判断 |
| 强制白天模式 | `{night=0}` | 关闭夜间告警判断 |
| 布防 | `{arm=2}` | 同夜间模式 |
| 撤防 | `{arm=0}` | 同白天模式 |
| 解除告警 | `{reset=1}` | 清除告警状态 + 关闭继电器 |
| 设置上报周期 | `{V0=10}` | 改为10秒上报一次 |

### 3.3 继电器控制

| 说明 | 命令 | 效果 |
|------|------|------|
| 开启继电器1 | `{OD1=64}` | D1 bit6 置1 |
| 关闭继电器1 | `{CD1=64}` | D1 bit6 清零 |
| 开启继电器2 | `{OD1=128}` | D1 bit7 置1 |
| 关闭继电器2 | `{CD1=128}` | D1 bit7 清零 |

### 3.4 自动上报字段

节点定时上报以下字段：

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `pir` | PIR人体红外(0=无人, 1=有人) | 0 |
| `tch` | 振动/门铃(0=无, 1=有) | 0 |
| `door` | 门磁状态(0=关, 1=开) | 0 |
| `flame` | 火焰(0=无, 1=有) | 0 |
| `gas` | 可燃气体 | 0 |
| `grating` | 光栅 | 0 |
| `alert` | 告警等级(0=正常, 1=注意, 2=报警) | 0 |
| `stay` | PIR持续触发时间(秒) | 0 |
| `night` | 夜间模式(0=白天, 1=夜间) | 0 |

### 3.5 告警触发条件

| 告警等级 | 触发条件 |
|----------|----------|
| 0 (正常) | 无异常 |
| 1 (注意) | PIR持续触发 >10秒 且未按门铃(徘徊检测) |
| 2 (报警) | 夜间模式 + 门被打开(入侵告警) |

## 4. Coordinator — 协调器

**节点地址**：`0x0000`  
**节点名称**：000  
**功能**：ZigBee网络管理 + 数据转发

### 4.1 查询命令

| 说明 | 命令 | 返回示例 |
|------|------|----------|
| 查询PAN ID | `{PANID=?}` | `{PANID:1234}` |
| 查询信道 | `{CHANNEL=?}` | `{CHANNEL:15}` |
| 查询节点类型 | `{TYPE=?}` | `{TYPE:12000}` |
| 回显测试 | `{ECHO=hello}` | `{ECHO:hello}` |

### 4.2 设置命令

| 说明 | 命令 | 效果 |
|------|------|------|
| 设置PAN ID | `{PANID=5678}` | 修改PAN ID并重启 |
| 设置信道 | `{CHANNEL=15}` | 修改信道(11-26)并重启 |

## 5. 测试步骤建议

### 5.1 基础测试（先测这个）

```
1. 智云平台发送到 sensor-b(0x0002)：{rgb=[255,0,0]}
   预期：RGB亮红灯

2. 智云平台发送到 sensor-b(0x0002)：{buzz=500}
   预期：蜂鸣器短响 + 蓝灯亮5秒

3. 智云平台发送到 sensor-b(0x0002)：{unlock=1}
   预期：继电器开 + 绿灯亮 + 3秒后自动关门
```

### 5.2 传感器查询测试

```
1. 智云平台发送到 sensor-a(0x0001)：{temp=?}
   预期：返回当前温度

2. 智云平台发送到 sensor-c(0x0003)：{pir=?}
   预期：返回PIR状态（手遮挡/离开测试）

3. 智云平台发送到 sensor-c(0x0003)：{door=?}
   预期：返回门磁状态（用磁铁靠近/远离测试）
```

### 5.3 告警联动测试

```
1. 发送到 sensor-c(0x0003)：{night=1}
   预期：启用夜间模式

2. 触发门磁（用磁铁模拟开门）

3. 观察 sensor-c 上报 alert=2

4. 手动发送到 sensor-b(0x0002)：{alert=2}
   预期：sensor-b 红灯 + 蜂鸣器长响

5. 发送到 sensor-b(0x0002)：{reset=1}
   预期：恢复正常
```

## 6. 常见问题

### Q1: 发送命令后没有反应？

检查：
- 节点是否已加入网络（LED2常亮）
- 目标地址是否正确
- 命令格式是否正确（花括号、等号）

### Q2: 查询返回空？

检查：
- 传感器是否正确连接
- 是否使用了正确的字段名

### Q3: 告警联动不工作？

说明：
- sensor-c 检测到告警后会自动上报 `alert` 字段
- 但 sensor-c 和 sensor-b 不能直接通信
- 需要通过 Python 上位机转发（或手动从智云平台下发 `{alert=N}` 到 sensor-b）

## 附录：D0/D1 位掩码参考

### D0（传感器上报使能位）

| Bit | 传感器 | 使能值 | 禁用值 |
|-----|--------|--------|--------|
| 0 | temp(温度) | OD0=1 | CD0=1 |
| 1 | humi(湿度) | OD0=2 | CD0=2 |
| 2 | lux(光照) | OD0=4 | CD0=4 |

### D1（执行器控制位，sensor-b/c）

| Bit | 执行器 | 开启 | 关闭 |
|-----|--------|------|------|
| 0 | RGB_R(红) | OD1=1 | CD1=1 |
| 1 | RGB_G(绿) | OD1=2 | CD1=2 |
| 2 | RGB_B(蓝) | OD1=4 | CD1=4 |
| 3 | 蜂鸣器 | OD1=8 | CD1=8 |
| 4 | LED1 | OD1=16 | CD1=16 |
| 5 | LED2 | OD1=32 | CD1=32 |
| 6 | 继电器1 | OD1=64 | CD1=64 |
| 7 | 继电器2 | OD1=128 | CD1=128 |
