# 给队友的通信接口说明

本文档用于统一智能门禁访客管理系统中各模块之间的数据字段、上传格式和下发命令。A/B/C/E 同学按本文档对接即可。

## 1. 总体数据链路

上行数据：

```text
硬件节点 -> ZXBee/ZigBee -> 协调器 -> 智云平台 -> Python/PyQt
```

下行控制：

```text
Python/PyQt -> 智云平台 -> 协调器 -> ZXBee/ZigBee -> 硬件节点
```

智云平台作为中转站，Python/PyQt 不直接连接硬件。

## 2. 节点分工

| 节点工程 | 节点名 | 负责人模块 | 主要字段 |
| --- | --- | --- | --- |
| `sensor-a` | `601` | 环境采集 | `temp`, `humi`, `lux` |
| `sensor-b` | `602` | 控制执行 | `unlock`, `buzz`, `rgb`, `reset` |
| `sensor-c` | `603` | 安防检测 | `pir`, `door`, `tch`, `alert` |

## 3. 统一字段表

### 3.1 上行字段

这些字段由硬件节点上传到智云平台，Python/PyQt 负责读取和显示。

| 字段 | 类型 | 取值 | 含义 | 来源节点 |
| --- | --- | --- | --- | --- |
| `pir` | int | `0/1` | 人体红外，0无人，1有人 | `sensor-c` |
| `door` | int | `0/1` | 门磁状态，0关门，1开门 | `sensor-c` |
| `tch` | int | `0/1` | 门铃/触摸按键，0未按，1按下 | `sensor-c` |
| `temp` | float | 实际温度 | 温度，单位摄氏度 | `sensor-a` |
| `humi` | float | `0-100` | 湿度，单位 `%RH` | `sensor-a` |
| `lux` | int | `0-65535` | 光照强度，单位 lux | `sensor-a` |
| `alert` | int | `0/1/2` | 告警级别，0正常，1注意，2报警 | `sensor-c` |
| `stay` | int | 秒数 | 人员停留时间 | `sensor-c` |
| `night` | int | `0/1` | 是否夜间事件，1表示夜间 | `sensor-c` |

### 3.2 下行字段

这些字段由 Python/PyQt 通过智云平台下发给硬件节点。

| 字段 | 类型 | 取值 | 含义 | 目标节点 |
| --- | --- | --- | --- | --- |
| `unlock` | int | `0/1` | 远程开门，1开门 | `sensor-b` |
| `buzz` | int | `0/1/500` 等 | 蜂鸣器控制，0关，1长响，500短响500ms | `sensor-b` |
| `rgb` | array | `[R,G,B]` | RGB颜色，三通道范围 `0-255` | `sensor-b` |
| `reset` | int | `0/1` | 解除告警/复位，1执行 | `sensor-b` / `sensor-c` |

## 4. 命令字

| CMD | 方向 | 说明 |
| --- | --- | --- |
| `0x01` | 硬件 -> 智云 | 定时状态/环境上报 |
| `0x02` | 硬件 -> 智云 | 安防事件立即上报 |
| `0x03` | 智云 -> 硬件 | 远程控制命令 |
| `0x06` | 智云 -> 硬件 | 告警解除/复位 |

## 5. 上行数据格式

### 5.1 环境数据上报

来源：`sensor-a`

用途：给 PyQt 环境信息面板显示。

```json
{"temp":27.0,"humi":65.0,"lux":200}
```

建议触发方式：

```text
定时上报，CMD=0x01
```

### 5.2 门禁状态上报

来源：`sensor-c`

用途：给 PyQt 门禁状态看板显示。

```json
{"pir":0,"door":0,"tch":0,"alert":0}
```

建议触发方式：

```text
定时上报或状态变化上报，CMD=0x01
```

### 5.3 门铃事件上报

来源：`sensor-c`

用途：生成访客通知和事件日志。

```json
{"tch":1,"door":0,"alert":0}
```

建议触发方式：

```text
门铃/触摸触发后立即上报，CMD=0x02
```

### 5.4 徘徊事件上报

来源：`sensor-c`

用途：生成注意级别安防事件。

```json
{"pir":1,"stay":15,"alert":1}
```

建议触发方式：

```text
PIR检测到有人停留超过10秒且未按门铃，CMD=0x02
```

### 5.5 夜间异常开门上报

来源：`sensor-c`

用途：生成报警级别安防事件。

```json
{"door":1,"night":1,"alert":2}
```

建议触发方式：

```text
夜间 22:00-6:00 门被打开且无授权，CMD=0x02
```

## 6. 下行控制格式

### 6.1 远程开门

来源：Python/PyQt

目标：`sensor-b`

```json
{"unlock":1,"rgb":[0,255,0]}
```

执行效果：

```text
模拟开门，RGB绿灯亮3秒
```

建议命令字：

```text
CMD=0x03
```

### 6.2 远程触发门铃

来源：Python/PyQt

目标：`sensor-b`

```json
{"buzz":500,"rgb":[0,0,255]}
```

执行效果：

```text
蜂鸣器短响500ms，RGB蓝灯闪烁
```

建议命令字：

```text
CMD=0x03
```

### 6.3 远程声光告警

来源：Python/PyQt

目标：`sensor-b`

```json
{"buzz":1,"rgb":[255,0,0]}
```

执行效果：

```text
蜂鸣器长响，RGB红灯报警
```

建议命令字：

```text
CMD=0x03
```

### 6.4 告警解除

来源：Python/PyQt

目标：`sensor-b` / `sensor-c`

```json
{"reset":1}
```

执行效果：

```text
关闭蜂鸣器，RGB恢复正常，alert恢复0
```

建议命令字：

```text
CMD=0x06
```

## 7. 给各位队友的对接要求

### 7.1 给 A 同学/安防检测模块

请提供或确认以下数据：

| 数据 | 字段 | 要求 |
| --- | --- | --- |
| 人体红外 | `pir` | 0无人，1有人 |
| 门磁状态 | `door` | 0关门，1开门 |
| 门铃/触摸 | `tch` | 0未按，1按下 |
| 告警级别 | `alert` | 0正常，1注意，2报警 |

需要支持的事件：

```text
门铃按下 -> {"tch":1,"door":0,"alert":0}
有人徘徊 -> {"pir":1,"stay":15,"alert":1}
夜间开门 -> {"door":1,"night":1,"alert":2}
```

### 7.2 给 B 同学/环境采集与数据库模块

请提供或确认以下数据：

| 数据 | 字段 | 要求 |
| --- | --- | --- |
| 温度 | `temp` | float，单位摄氏度 |
| 湿度 | `humi` | float，单位 `%RH` |
| 光照 | `lux` | int，单位 lux |

推荐环境上报：

```json
{"temp":27.0,"humi":65.0,"lux":200}
```

数据库建议使用同样字段保存，避免 Python 端二次改名。

### 7.3 给 C 同学/控制与联动模块

请提供或确认以下控制函数：

| 控制动作 | 字段 | 期望效果 |
| --- | --- | --- |
| 远程开门 | `unlock=1` | 模拟门锁打开 |
| 蜂鸣器短响 | `buzz=500` | 短响500ms |
| 蜂鸣器长响 | `buzz=1` | 持续报警 |
| RGB绿灯 | `rgb=[0,255,0]` | 正常/开门 |
| RGB蓝灯 | `rgb=[0,0,255]` | 门铃 |
| RGB红灯 | `rgb=[255,0,0]` | 告警 |
| 告警解除 | `reset=1` | 关闭报警并恢复状态 |

### 7.4 给 E 同学/Python PyQt 模块

Python/PyQt 需要按这些字段显示：

| 界面区域 | 使用字段 |
| --- | --- |
| 门禁状态看板 | `pir`, `door`, `tch`, `alert` |
| 环境信息面板 | `temp`, `humi`, `lux` |
| 访客时间线 | `tch`, `pir`, `door`, `alert`, `create_time` |
| 远程控制区 | `unlock`, `buzz`, `rgb`, `reset` |

Python/PyQt 下发控制时请使用：

```json
{"unlock":1,"rgb":[0,255,0]}
{"buzz":500,"rgb":[0,0,255]}
{"buzz":1,"rgb":[255,0,0]}
{"reset":1}
```

## 8. 智云平台字段配置

智云平台建议添加以下字段：

```text
pir
door
tch
temp
humi
lux
alert
stay
night
unlock
buzz
rgb
reset
```

如果智云平台不方便配置数组字段 `rgb`，可以拆成：

```text
rgb_r
rgb_g
rgb_b
```

对应：

```json
{"rgb_r":255,"rgb_g":0,"rgb_b":0}
```

## 9. 最终统一约定

全组统一使用：

```text
pir, door, tch, temp, humi, lux, alert, stay, night, unlock, buzz, rgb, reset
```

不要再在 Python、数据库或智云平台里继续使用 `A0/A1/A2/D1` 作为最终业务字段。`A0/A1/A2/D1` 可以作为硬件工程内部旧字段，但上传到智云和给 Python 使用时，应转换成上面的项目字段。
